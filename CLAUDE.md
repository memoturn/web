# CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## What this repo is

Web surfaces for Memoturn (the agent-memory database), split out of the product repo `memoturn/memoturn`. pnpm + Turborepo monorepo deployed to Cloudflare Workers:

- `apps/web` тАФ marketing site (TanStack Start + Vite, React 19) тЖТ memoturn.ai
- `apps/docs` тАФ docs (Astro Starlight) тЖТ docs.memoturn.ai
- `packages/ui` тАФ shared design system (shadcn new-york-v4 style, Tailwind v4, CVA), consumed as ESM source тАФ no build step
- `packages/tsconfig` тАФ shared TS configs: `dashboard.json` (apps/web), `docs.json` (apps/docs), `base.json`/`library.json` (packages), `worker.json`

## Commands

```bash
pnpm install
pnpm dev            # both apps (marketing on :3000 + docs)
pnpm dev:web        # marketing only (vite dev --port 3000)
pnpm dev:docs       # docs only (astro dev)
pnpm build          # turbo build of both apps
pnpm typecheck      # tsc --noEmit / astro check per package
pnpm check          # biome check --write (lint + format)
pnpm check:ci       # biome ci, no writes
pnpm run deploy     # build + test + wrangler deploy (both apps) тАФ "run" required, bare `pnpm deploy` hits pnpm's built-in
pnpm deploy:staging # staging workers (memoturn-web-staging / memoturn-docs-staging)
```

Scope to one package with `pnpm --filter @memoturn/web <script>` (also `@memoturn/docs`, `@memoturn/ui`). After changing `wrangler.jsonc`, regenerate worker types with `pnpm --filter <app> cf-typegen`. Turbo defines `test` / `test:integration` tasks, but no test files or vitest configs exist yet.

## Linting / formatting

Biome, not eslint/prettier (`biome.jsonc`): tabs, line width 140, double quotes. Generated files (`routeTree.gen.ts`, `worker-configuration.d.ts`) and the design-system CSS files are excluded тАФ don't "fix" formatting in those. A11y rules are deliberately relaxed under `packages/ui/src/components/` (shadcn primitives).

## Source-of-truth docs (read before touching copy or UI)

`BRAND.md` (voice + palette + motion), `DESIGN.md`/`DESIGN.json` (visual system, anti-references, accessibility floor), and `PRODUCT.md` (positioning, personas, voice rules per surface) are authoritative for all copy and visual decisions. `.impeccable.md` is only a pointer to them тАФ never edit it. Key hard rules:

- Design tokens live in `packages/ui/src/styles/tokens.css` (semantic CSS custom properties); no one-off colors or radii in components.
- Voice is technical, direct, declarative: no hype words ("supercharge", "unlock"), no exclamation marks, no emojis on product UI.
- Anti-references in DESIGN.md (gradient-hero SaaS, neon-purple AI, brutalist monospace, enterprise navy, orchestrator glow) mean: don't introduce those aesthetics.
- WCAG 2.2 AA; honor `prefers-reduced-motion`.
- All product copy describes the agent-memory database; positioning comes from `PRODUCT.md`, not from the product repo's docs.

## Architecture notes

**apps/web** тАФ TanStack Start file-based routing in `src/routes/` (`__root.tsx` layout + pages); `src/routeTree.gen.ts` is generated by the router plugin тАФ never hand-edit. `src/server.ts` wraps the Start handler with CSP (nonce-based) and origin checks. External URLs (docs, GitHub) come from `src/lib/public-urls.ts` via env, not hardcoded strings. Path alias `#/*` тЖТ `./src/*`.

**apps/docs** тАФ Starlight content in `src/content/docs/` (markdown), sidebar defined in `astro.config.mjs`, custom Astro components in `src/components/`. Deployed as static assets via wrangler (no worker script). Staging build needs `MEMOTURN_DOCS_URL` (see the `deploy:staging` script).

**packages/ui** тАФ `src/components/` holds shadcn primitives, `src/composite/` holds brand-level components (`BrandMark`, `CodeBlock`, `EmptyState`, `ThemeToggle`), `src/lib/utils.ts` exports `cn()`. Styles split: `tokens.css` (theme), `base.css` (global layer + brand utility classes like `.island-shell`, `.island-kicker`), `index.css` (aggregate import). Consumed via subpath exports (`@memoturn/ui/components/*`, `@memoturn/ui/styles/tokens.css`, тАж).

**Dependencies** are pinned through the pnpm catalog in `pnpm-workspace.yaml` тАФ add/update versions there and reference them as `"catalog:"` in package.json files.

**docker-compose.yml** is not a standalone stack: it's a dev overlay for the product repo's control-plane that bind-mounts a host-built console (`apps/console` doesn't exist in this repo yet); run it from the product repo root per the comments in the file.
